# ADR 0004: mDNS / Zeroconf dependency for LAN announce

- Status: Accepted
- Date: 05/06/2026
- Deciders: project owner
- Supersedes: —

## Context

The single biggest first-run friction point is the console-setup step: the user
must discover the PC's LAN IP and type it into the PS5/PS4 proxy settings. The
[roadmap](../roadmap.md) lists an mDNS announce (`psxdh._http._tcp`) as a Phase 4
enhancement so a `psxdh` instance is discoverable on the LAN ("psxdh found at
192.168.1.42").

Per [ADR 0002](0002-dependency-budget.md), any new external dependency requires
its own ADR before the import lands. Implementing mDNS/DNS-SD from scratch on
top of `golang.org/x/net` multicast sockets is a meaningful amount of
fiddly, platform-specific code (IPv4 + IPv6 multicast group membership,
duplicate-name probing, goodbye packets); a small, well-scoped library is the
better trade.

## Decision

Adopt **`github.com/grandcat/zeroconf`** for mDNS service registration.

- **Gap filled:** multicast DNS-SD registration + the platform multicast socket
  handling that stdlib does not provide.
- **Alternatives considered:**
  - `github.com/hashicorp/mdns` — capable, but licensed **MPL-2.0**, which is
    outside the "MIT/BSD/Apache-2.0" allowlist in ADR 0002. Rejected on licence.
  - Hand-rolled on `golang.org/x/net/ipv4`/`ipv6` — most code to own and test;
    deferred unless the dependency proves problematic.
- **Licence:** zeroconf is **MIT**; its transitive `github.com/miekg/dns` is
  **BSD-3-Clause**; `golang.org/x/net` is already in the tree. All compatible
  with the project's MIT licence and the ADR 0002 policy.
- **Maintenance signal:** widely used (Hashicorp/Consul ecosystem siblings,
  many dependents), stable API surface (`Register` / `Server.Shutdown`).

The announce is **opt-in** (`mdns.enabled: false` by default) and lives behind
the thin `internal/mdns` wrapper so the dependency is isolated to one package
and can be swapped without touching the proxy command.

## Consequences

- `go.mod` gains `github.com/grandcat/zeroconf` (+ `github.com/miekg/dns`
  transitively). Both are MIT/BSD and reviewable.
- mDNS code is confined to `internal/mdns`; everything else depends on a 3-method
  interface (`Announce` / `Close`).
- If the dependency later rots, the wrapper boundary means a hand-rolled
  replacement is a localised change.
